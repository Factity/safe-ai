Transformer circuits, traced by hand

How does a stack of attention heads turn a token into a prediction? Start from nothing — a model with zero layers — and add just enough machinery to explain the strangest trick a two-layer transformer can pull: an induction head completing a pattern it has never explicitly been told about. Every diagram below is live — click, hover, and step through it.

The residual stream

Every attention layer and every MLP in a transformer reads from one shared channel — the residual stream — and writes back into it by simple addition. Nothing is ever erased or overwritten; each layer just adds its own correction on top of whatever is already there.

xfinal = embed(token) + attn0(x) + mlp0(x) + attn1(x) + mlp1(x) + ...

Because the stream is a running sum, the whole computation is linear at the top level — which means the final logits can always be decomposed back into independent, additive contributions from each layer. That single fact is what makes everything else on this page possible.

Zero layers: just bigrams

Delete every attention layer and every MLP, and only the direct path is left: an embedding matrix feeding straight into an unembedding matrix.

logits = WU WE · token

That product W_U W_E is a fixed matrix of size vocab × vocab. Row i, column j is exactly the log-likelihood the model assigns to token j following token i — a bigram table, nothing more. There's no attention, so no notion of context beyond "what is the current token."

Every cell is a fixed number learned from corpus statistics — WUWE never looks left of the current token. Click the, cat, or any row label to preview its row.

This direct path never disappears as we add layers — it's still sitting there in every deeper model, just joined by additional terms. Adding a single attention head is what lets the model finally condition on something other than the current token.

One layer: QK decides where, OV decides what

A single attention head factors cleanly into two independent circuits that never interact with each other:

WQK = WQT WK — the QK circuit: decides the attention pattern (where to look)
WOV = WV WO — the OV circuit: decides what gets copied once attention has picked a source

Because these are separate matrices, the exact same attention pattern could be paired with a completely different OV circuit to do a different job. Below, a toy duplicate-token head: its QK circuit fires when the key token matches the query token, and its OV circuit boosts the logit for that same token — a head that predicts repetition.

Two layers: composition and induction heads

A second layer can read a first layer's output, not just the raw embedding — this is called composition. When a layer-1 head builds its keys from a layer-0 head's output, that's K-composition, and it's the mechanism behind one of the most important circuits ever found in a language model: the induction head.

Induction heads implement one rule: if this exact token appeared before, predict whatever followed it last time. It's the circuit that lets a model complete patterns, copy names, and follow in-context examples — a primitive form of in-context learning, built from nothing more than two attention heads stacked on top of each other.

Induction head, step by step
sequence: D E F D — what comes after the second D?
Step 0. The token D repeats at position 3. Last time it appeared, it was followed by E. Can the model recover that?

Expanding the full equation

Because the residual stream is additive, an attention-only transformer's logits expand into a sum of fully independent, human-readable paths. Click a term to unpack it.

Logits = Σ (paths)
Direct path
WU WE
Layer-0 head paths
Σh WU WOV0,h A0,h WE
Layer-1 head paths
Σh WU WOV1,h A1,h WE
Virtual head paths
Σh0,h1 WU WOV1,h1 A1,h1(A0,h0WOV0,h0) WE
Direct path. Skips every layer entirely — this is exactly the zero-layer bigram table from section two. It's still part of the sum in every deeper model.

The last term — virtual head paths — is where induction heads live: a layer-1 head's attention pattern is computed using a layer-0 head's output instead of the raw token, so the pair of heads behaves like one new, more powerful "virtual" attention head. Real models drop plenty of terms here too: MLP layers and three-or-more-layer compositions multiply out combinatorially, which is a big part of why interpretability gets so much harder as models grow.
