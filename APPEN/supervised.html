Supervised learning is the branch of machine learning concerned with learning a mapping from inputs to outputs based on example input–output pairs. It encompasses regression (predicting continuous values), classification (predicting discrete categories), and structured prediction. This reference builds from first principles: the definition of the learning problem and key concepts like loss functions and generalization (B1.1–B1.19), then progresses through linear models, tree‑based methods, kernel methods, neural networks, ensemble techniques, probabilistic approaches, and model evaluation (B1.20–B1.130), with each topic presented as a formal definition, theorem, or algorithm accompanied by conceptual explanation.
B1.1

Supervised Learning (Definition)

Supervised learning is the task of learning a function f : 𝒳 → 𝒴 from a labeled dataset 𝒟 = {(xi, yi)}i=1n, where each xi ∈ 𝒳 is a feature vector and yi ∈ 𝒴 is its corresponding label or target value. The goal is to produce a model f̂ that generalizes well: for a new, unseen input x, the prediction f̂(x) should be close to the true y.

  • 𝒳the input (feature) space, typically ℝd or a subset.
  • 𝒴the output (label) space; ℝ for regression, {0,1} or {1,…,K} for classification.
  • f : 𝒳 → 𝒴the unknown true function we aim to approximate.
B1.2

Features and Labels

A feature (or predictor, attribute, independent variable) is an individual measurable property of the phenomenon being observed. The label (or target, response, dependent variable) is the quantity we wish to predict. Features are typically represented as a vector x ∈ ℝd, where d is the number of features (dimensionality).

  • x(j)the j‑th feature of input x, j = 1,…,d.
  • ythe label associated with input x.
B1.3

Training, Validation, and Test Sets

The available labeled data is typically partitioned into three disjoint subsets:

  • Training set: used to fit the model parameters.
  • Validation set: used for hyperparameter tuning and model selection; not used for training.
  • Test set: held out until final evaluation; provides an unbiased estimate of generalization performance.

A common split ratio is 60/20/20 or 70/15/15, depending on dataset size.

B1.4

Hypothesis Space

The hypothesis space ℋ is the set of all functions h : 𝒳 → 𝒴 that the learning algorithm is allowed to select from. For linear regression, ℋ = {h(x) = wTx + b ∣ w ∈ ℝd, b ∈ ℝ}. The choice of ℋ encodes the inductive bias of the learner — its built‑in assumptions about the nature of the target function.

  • ℋthe hypothesis space; the set of candidate functions considered by the learner.
B1.5

Loss Function

A loss function ℓ : 𝒴 × 𝒴 → ℝ≥0 quantifies the discrepancy between a predicted value ŷ and the true label y. Common examples:

ℓsquared(y, ŷ) = (y − ŷ)2      (squared error, for regression)
ℓ0-1(y, ŷ) = 1[y ≠ ŷ]       (zero‑one loss, for classification)
ℓhinge(y, ŷ) = max(0, 1 − y·ŷ)    (hinge loss, for SVMs)
ℓlog(y, ŷ) = −y log(ŷ) − (1−y) log(1−ŷ) (binary cross‑entropy)
  • ℓ(y, ŷ)the loss incurred when predicting ŷ while the true label is y.
B1.6

Risk (Expected Loss / Generalization Error)

The risk (or expected loss) of a hypothesis h is the expected loss over the true data distribution P(x, y):

R(h) = 𝔼(x,y)∼P[ℓ(y, h(x))].

This is the quantity we truly wish to minimize, but it is inaccessible because P is unknown. The risk quantifies generalization — performance on future, unseen data.

B1.7

Empirical Risk

The empirical risk is the average loss on the training set 𝒟:

R̂n(h) = (1/n) ∑i=1n ℓ(yi, h(xi)).

Empirical Risk Minimization (ERM) is the principle of choosing h that minimizes R̂n. This is what virtually all supervised learning algorithms do, either exactly (OLS) or approximately (gradient descent).

B1.8

Regression vs Classification

  • Regression: 𝒴 is continuous (typically ℝ). The goal is to predict a real‑valued quantity. Examples: predicting house prices, temperature, stock returns.
  • Classification: 𝒴 is discrete (finite set of categories). The goal is to assign an input to one of K classes. Examples: spam detection (K=2), handwritten digit recognition (K=10).
The distinction is not always sharp — ordinal regression, ranking, and structured prediction bridge the gap.
B1.9

Binary and Multi‑class Classification

In binary classification, 𝒴 = {0,1} (or {−1,+1}). Many algorithms are natively binary (logistic regression, SVM). For multi‑class classification with K > 2 classes, strategies include:

  • One‑vs‑Rest (OvR): train K binary classifiers, each distinguishing one class from all others.
  • One‑vs‑One (OvO): train K(K−1)/2 binary classifiers for every pair of classes.
  • Softmax / multinomial: a single model with K outputs normalized by the softmax function (B1.31).
B1.10

Training Error vs Test Error

The training error is the empirical risk on the training set; the test error is the empirical risk on the held‑out test set. A large gap between them signals overfitting: the model has memorized the training data rather than learning generalizable patterns. A model with high training error and high test error is underfitting.

B1.11

Generalization

Generalization is the ability of a model to perform accurately on unseen data. The generalization gap is R(h) − R̂n(h). PAC (Probably Approximately Correct) learning theory provides bounds on this gap in terms of the complexity of ℋ (e.g., VC dimension) and the sample size n.

ML relevance: all of model selection, regularization, and cross‑validation are strategies to minimize the generalization gap.
B1.12

Overfitting and Underfitting

  • Overfitting: the model is too complex relative to the amount/quality of training data; it captures noise as if it were signal. Characterized by low training error, high test error.
  • Underfitting: the model is too simple to capture the underlying structure. Characterized by high training error and high test error.

The bias–variance tradeoff (B1.13) provides a formal decomposition of this phenomenon.

B1.13

Bias–Variance Decomposition

For squared‑error loss, the expected prediction error at a point x can be decomposed as:

𝔼[(y − f̂(x))2] = (Bias[f̂(x)])2 + Var[f̂(x)] + σ2

where σ2 is the irreducible noise, Bias measures how far the average prediction is from the truth, and Variance measures how much predictions fluctuate across different training sets. Flexible models have low bias but high variance; rigid models have high bias but low variance. The tradeoff is fundamental to model selection.

B1.14

No Free Lunch Theorem

The No Free Lunch (NFL) theorem states that, averaged over all possible data distributions, every learning algorithm has the same expected performance on unseen data. There is no universally best algorithm. The practical implication: assumptions (inductive bias) are necessary; the art of ML is matching the algorithm's bias to the structure of the problem.

B1.15

Inductive Bias

Inductive bias is the set of assumptions a learning algorithm makes about the target function. Examples:

  • Linear regression: the target is linear in the features.
  • k‑NN: nearby points have similar labels (smoothness).
  • Decision trees: the target can be expressed as axis‑aligned splits.
  • Neural networks: the target is a composition of simple non‑linear functions.

The inductive bias is what allows an algorithm to generalize beyond the training data.

B1.16

Model Complexity

Model complexity refers to the capacity of the hypothesis space — roughly, how many different functions it can represent. Measured by:

  • Number of parameters (e.g., polynomial degree in regression).
  • VC dimension (for binary classifiers).
  • Rademacher complexity.
  • Effective degrees of freedom.

Higher complexity allows fitting more intricate patterns but increases the risk of overfitting.

B1.17

Parametric vs Non‑parametric Models

  • Parametric models: have a fixed, finite number of parameters regardless of dataset size. Examples: linear regression, logistic regression, neural networks (fixed architecture). Fast to train, but may underfit if the parameter count is insufficient.
  • Non‑parametric models: the effective number of parameters grows with the data. Examples: k‑NN, kernel regression, Gaussian processes. More flexible, but can be computationally expensive and prone to overfitting without careful regularization.
B1.18

Notation Conventions

Throughout this reference:

  • Scalars: italic lowercase (y, α, n).
  • Vectors: bold lowercase (x, w) or italic with superscript (x(j)).
  • Matrices: bold uppercase (X, W).
  • Design matrix X ∈ ℝn×d: rows are training examples, columns are features.
  • Label vector y ∈ ℝn: the corresponding labels.
  • Prediction: ŷ = f̂(x).
B1.19

Design Matrix and Label Vector

The design matrix X ∈ ℝn×d stacks n training examples as rows, each with d features:

X = [x1T; x2T; …; xnT] with y = (y1, y2, …, yn)T.

This compact notation is used throughout: the i-th row of X is xiT, and the j-th column contains all values of feature j across the dataset.

B1.20

Linear Regression

Linear regression models the target as a linear function of the features:

f̂(x) = wTx + b = ∑j=1d wjx(j) + b.

The parameters w ∈ ℝd (weights) and b ∈ ℝ (bias or intercept) are learned from data. Despite its simplicity, linear regression is a foundational building block: it underlies more complex models (polynomial regression via basis expansion, the final layer of many neural networks, etc.).

ML relevance: linear regression is the first model tried in virtually every regression task — it is fast, interpretable, and provides a baseline for more complex approaches.
B1.21

Ordinary Least Squares (OLS)

Ordinary Least Squares finds the parameters that minimize the sum of squared errors:

minw,b ∑i=1n (yi − wTxi − b)2 = ∥y − Xw − b·1n∥2.

This is the maximum likelihood estimator under the assumption of Gaussian noise with constant variance.

B1.22

Normal Equations

Assuming the bias is absorbed into w by prepending a constant feature of 1 to each xi, the OLS solution is given by the normal equations:

XTX w = XTy ⇒ w* = (XTX)−1XTy,

provided XTX is invertible (i.e., X has full column rank). The matrix (XTX)−1XT is the Moore–Penrose pseudoinverse of X when X is full‑rank.

B1.23

Geometric Interpretation of OLS

Geometrically, OLS projects the label vector y onto the column space of X. The prediction ŷ = Xw* is the orthogonal projection of y onto span(columns of X), and the residual vector y − ŷ is orthogonal to every column of X: XT(y − ŷ) = 0. This is why it is called "least squares" — it minimizes the Euclidean distance between y and the subspace spanned by the features.

B1.24

Gauss–Markov Theorem

Under the assumptions that errors are uncorrelated, have zero mean, and have equal variance (homoscedasticity), the OLS estimator is the Best Linear Unbiased Estimator (BLUE): among all linear unbiased estimators, it has the smallest variance. No distributional assumption (e.g., normality) is needed for this result.

B1.25

Ridge Regression (L2 Regularization)

Ridge regression adds an L2 penalty on the weights to the OLS objective:

minw ∥y − Xw∥2 + λ∥w∥2,

with closed‑form solution w* = (XTX + λI)−1XTy. The penalty λ ≥ 0 shrinks weights toward zero, improving numerical stability (always invertible, even when XTX is singular) and reducing variance at the cost of introducing bias. This is a classic realization of the bias–variance tradeoff.

B1.26

Lasso Regression (L1 Regularization)

Lasso (Least Absolute Shrinkage and Selection Operator) uses an L1 penalty:

minw ∥y − Xw∥2 + λ∥w∥1.

Unlike ridge, the L1 penalty drives some coefficients exactly to zero, performing automatic feature selection. The solution is not available in closed form; it is found via convex optimization (coordinate descent, proximal gradient methods). Lasso is especially valuable in high‑dimensional settings (d ≫ n).

B1.27

Elastic Net

Elastic Net combines L1 and L2 penalties:

minw ∥y − Xw∥2 + λ1∥w∥1 + λ2∥w∥2.

It inherits Lasso's sparsity (feature selection) while addressing some of its limitations: when features are highly correlated, Lasso tends to pick one arbitrarily; Elastic Net tends to select groups of correlated features together.

B1.28

Logistic Regression

Logistic regression is a linear model for binary classification. It models the probability that an input belongs to the positive class as:

P(y=1 ∣ x) = σ(wTx + b) = 1 / (1 + e−(wTx + b)),

where σ(z) = 1/(1+e−z) is the sigmoid (logistic) function, mapping ℝ → (0,1). The decision boundary is linear: predict y=1 if wTx + b > 0.

B1.29

Sigmoid (Logistic) Function

The sigmoid function σ(z) = 1/(1 + e−z) has several important properties:

  • σ(z) ∈ (0,1) for all z ∈ ℝ.
  • σ(0) = 0.5.
  • It is monotonic and S‑shaped.
  • Its derivative is σ′(z) = σ(z)(1−σ(z)), a convenient form for gradient‑based optimization.
  • Its inverse is the logit function: logit(p) = log(p/(1−p)).
B1.30

Binary Cross‑Entropy Loss

For binary classification, the cross‑entropy loss (log loss) for a single example is:

ℓ(y, ŷ) = −[y log(ŷ) + (1−y) log(1−ŷ)],

where ŷ = σ(wTx + b). This is the negative log‑likelihood under the Bernoulli model. Minimizing cross‑entropy is equivalent to maximum likelihood estimation for logistic regression. The loss is convex in w, guaranteeing that gradient descent finds the global optimum.

B1.31

Softmax (Multinomial Logistic) Regression

For K-class classification, softmax regression generalizes logistic regression:

P(y=k ∣ x) = exp(wkTx + bk) / ∑j=1K exp(wjTx + bj).

The softmax function maps a vector of K raw scores (logits) to a probability distribution over the K classes. The corresponding loss is categorical cross‑entropy: ℓ = −∑k yk log(ŷk), where y is a one‑hot encoded true label.

B1.32

Maximum Likelihood Estimation (MLE)

Maximum Likelihood Estimation chooses parameters θ that maximize the probability (likelihood) of observing the training data under the model. For i.i.d. data:

θMLE = argmaxθ ∏i=1n P(yi ∣ xi; θ).

Equivalently, minimize the negative log‑likelihood. OLS corresponds to MLE with Gaussian noise; logistic regression is MLE under the Bernoulli model. MLE is consistent (converges to the true parameter as n → ∞) under mild regularity conditions.

B1.33

Gradient Descent

Gradient descent is an iterative optimization algorithm that updates parameters in the direction of steepest descent of the loss function:

θ(t+1) = θ(t) − η ∇θJ(θ(t)),

where η > 0 is the learning rate (step size) and J(θ) is the objective (typically empirical risk). For convex objectives (OLS, logistic regression), gradient descent converges to the global minimum. For non‑convex objectives (neural networks), it finds a local minimum.

B1.34

Stochastic Gradient Descent (SGD)

SGD replaces the full gradient (over all n examples) with an estimate from a single randomly chosen example (or a mini‑batch):

θ(t+1) = θ(t) − η ∇θℓ(yi, f̂(xi; θ)).

The gradient is noisy but unbiased. SGD is vastly more scalable for large datasets, introduces stochasticity that can help escape shallow local minima, and is the workhorse of modern deep learning.

B1.35

Mini‑batch Gradient Descent

Mini‑batch GD is a compromise between full‑batch GD and pure SGD: each step uses a small random subset (mini‑batch) of size m (typically 32–512):

θ(t+1) = θ(t) − (η/m) ∑i∈ℬt ∇θℓ(yi, f̂(xi; θ)).

Benefits include reduced variance in the gradient estimate (vs. pure SGD), computational efficiency via vectorized operations, and compatibility with GPU hardware.

B1.36

Learning Rate and Scheduling

The learning rate η is the most critical hyperparameter in gradient‑based optimization. Too large: divergence or oscillation. Too small: slow convergence. Common scheduling strategies include:

  • Step decay: reduce η by a factor every k epochs.
  • Exponential decay: ηt = η0 · γt.
  • Cosine annealing: smoothly decrease η following a cosine curve.
  • Adaptive methods: Adam, RMSProp, AdaGrad — each parameter gets its own learning rate based on historical gradients.
B1.37

Convergence of Gradient Descent for Convex Functions

For an L-smooth, convex function J, gradient descent with constant step size η ≤ 1/L converges at a rate of O(1/t). For μ-strongly convex functions, the rate improves to linear convergence: J(θ(t)) − J* ≤ (1 − ημ)t. The condition number κ = L/μ governs the practical convergence speed — poorly conditioned problems (large κ) converge slowly.

B1.38

Perceptron

The perceptron is the simplest neural model for binary classification (y ∈ {−1,+1}):

f̂(x) = sign(wTx + b).

The perceptron algorithm updates weights only when a mistake is made: if yi(wTxi) ≤ 0, set w ← w + η·yixi. It is an online algorithm — it processes examples one at a time and converges to a separating hyperplane if the data is linearly separable.

B1.39

Perceptron Convergence Theorem

If the training data is linearly separable with margin γ > 0 and bounded by radius R (i.e., ∥xi∥ ≤ R), the perceptron algorithm makes at most (R/γ)2 mistakes before finding a separating hyperplane — regardless of the order in which examples are presented. This guarantees convergence in finitely many steps.

B1.40

Multi‑layer Perceptron (MLP)

An MLP (feedforward neural network) is composed of multiple layers of neurons, each applying a linear transformation followed by a non‑linear activation:

h(l) = φ(l)(W(l)h(l−1) + b(l)),

where h(0) = x, φ is an activation function, and the final layer produces the output. MLPs can approximate any continuous function on compact sets (Universal Approximation Theorem, B1.43).

B1.41

Activation Functions

Common activation functions:

  • ReLU: φ(z) = max(0, z). Simple, avoids vanishing gradients for positive inputs, but can "die" for negative inputs.
  • Sigmoid: φ(z) = 1/(1+e−z). Smooth, bounded, but saturates → vanishing gradients.
  • Tanh: φ(z) = tanh(z). Zero‑centered, but also saturates.
  • Leaky ReLU: φ(z) = max(αz, z) with small α. Fixes the dying ReLU problem.
  • GELU: smooth approximation to ReLU, popular in transformers.
B1.42

Backpropagation

Backpropagation is the algorithm for computing gradients in a neural network efficiently. It applies the chain rule of calculus recursively from the output layer backward through the network. For a network with L layers, backprop computes ∂ℓ/∂W(l) and ∂ℓ/∂b(l) for all l in O(L) time, reusing intermediate results (the "error" signals) to avoid redundant computation.

Backpropagation is not a training algorithm — it is the gradient computation engine that enables gradient descent to train neural networks.
B1.43

Universal Approximation Theorem

A feedforward neural network with a single hidden layer containing a finite number of neurons, using a non‑constant, bounded, continuous activation function (e.g., sigmoid), can approximate any continuous function on a compact subset of ℝd to arbitrary accuracy. This is a theoretical existence result — it does not guarantee that training will find such an approximation, nor does it specify how many neurons are needed.

B1.44

Regularization in Neural Networks

Techniques to prevent overfitting in neural networks include:

  • L2 regularization (weight decay): add λ∑∥W(l)∥2 to the loss.
  • L1 regularization: encourages sparse weights.
  • Early stopping: halt training when validation error starts rising.
  • Dropout: randomly zero out neurons during training (B1.45).
  • Batch normalization: normalizes activations within each mini‑batch, providing a mild regularizing effect and enabling higher learning rates.
  • Data augmentation: artificially expand the training set with transformations (rotations, flips, noise).
B1.45

Dropout

Dropout is a regularization technique: during each training step, each neuron is independently "dropped" (set to zero) with probability p. At test time, all neurons are used, but their outputs are scaled by 1−p (or equivalently, weights are scaled). Dropout can be interpreted as training an ensemble of exponentially many subnetworks that share parameters, providing a powerful, cheap form of model averaging.

B1.46

Decision Trees

A decision tree recursively partitions the feature space into axis‑aligned regions and assigns a constant prediction to each region. Internal nodes test a feature against a threshold; leaf nodes contain predictions. Trees are interpretable, handle mixed data types, require little preprocessing, and are the foundation of powerful ensemble methods (random forests, gradient boosting).

B1.47

Splitting Criteria

At each node, the tree selects the feature and threshold that maximize the "purity" improvement. For regression, the criterion is typically variance reduction: split to minimize the weighted sum of child‑node variances. For classification, common criteria include information gain (based on entropy), Gini impurity, and misclassification rate.

B1.48

Information Gain

Information gain is the reduction in entropy achieved by a split:

IG(S, A) = H(S) − ∑v∈Values(A) (|Sv|/|S|) · H(Sv),

where H(S) = −∑k pk log2(pk) is the entropy of the class distribution at node S, and Sv is the subset of examples where feature A takes value v. The split with highest IG is chosen.

B1.49

Gini Impurity

Gini impurity measures the probability of misclassifying a randomly chosen element if it were labeled according to the class distribution at the node:

G(S) = 1 − ∑k=1K pk2.

Gini is computationally cheaper than entropy (no logarithms) and behaves similarly in practice. It is the default splitting criterion in CART (Classification And Regression Trees).

B1.50

Entropy

Entropy H(S) quantifies the uncertainty (impurity) of the class distribution at a node. H = 0 when all examples belong to one class (pure node); H is maximized when classes are equally represented. Entropy‑based splitting (used in ID3 and C4.5) tends to produce more balanced trees than Gini, but at slightly higher computational cost.

B1.51

CART Algorithm

CART (Classification And Regression Trees) builds binary trees by greedily choosing, at each node, the feature and threshold that minimize the weighted impurity of the two child nodes. The tree is grown to a maximum size and then pruned back using cost‑complexity pruning to avoid overfitting. CART handles both regression (squared error criterion) and classification (Gini impurity).

B1.52

Pruning

Pruning reduces tree complexity to improve generalization. Pre‑pruning (early stopping) halts growth when a node's impurity improvement is below a threshold or when the node has too few samples. Post‑pruning (e.g., cost‑complexity pruning) grows a full tree, then removes subtrees that do not significantly reduce error on a validation set, trading a small increase in training error for a simpler model.

B1.53

Random Forest

A random forest is an ensemble of decision trees, each trained on a bootstrap sample of the data. Two sources of randomness reduce correlation among trees and improve generalization:

  • Bagging (bootstrap aggregating): each tree is trained on a random sample (with replacement) of the training data.
  • Feature randomness: at each split, only a random subset of m features (typically m ≈ √d for classification, m ≈ d/3 for regression) is considered.

Prediction is by majority vote (classification) or averaging (regression).

B1.54

Bagging (Bootstrap Aggregating)

Bagging is a general ensemble technique: train B base learners on B bootstrap samples from the original training set, then aggregate predictions (averaging for regression, voting for classification). Bagging reduces variance without increasing bias, and is most effective for high‑variance, low‑bias base learners like deep decision trees.

B1.55

Out‑of‑Bag (OOB) Error

For each bootstrap sample, roughly one‑third of the training examples are left out (out‑of‑bag). The OOB error estimates generalization performance by evaluating each example only on trees for which it was OOB, then averaging. OOB error is a free, unbiased alternative to cross‑validation for bagged ensembles like random forests.

B1.56

Feature Importance

Tree‑based models provide natural feature importance scores. Two common measures:

  • Mean decrease in impurity (MDI): total reduction in the splitting criterion (Gini, entropy, variance) attributed to each feature, averaged over all trees.
  • Permutation importance: the drop in model performance when a feature's values are randomly shuffled, breaking its association with the target.

Feature importance aids interpretability and feature selection.

B1.57

Gradient Boosting

Gradient boosting builds an ensemble sequentially: each new tree is trained to predict the negative gradient (residuals) of the loss function with respect to the current ensemble's predictions. Formally, at iteration m, with current model Fm−1, the new tree hm fits the pseudo‑residuals:

ri = −[∂ℓ(yi, F(xi)) / ∂F(xi)]F=Fm−1.

Then Fm(x) = Fm−1(x) + ν·hm(x), where ν (learning rate, shrinkage) controls the contribution of each tree.

B1.58

AdaBoost

AdaBoost (Adaptive Boosting) is the original boosting algorithm for binary classification. It sequentially trains weak learners (typically shallow trees), reweighting training examples at each step: misclassified examples receive higher weight, forcing subsequent learners to focus on hard cases. The final prediction is a weighted majority vote of all learners, with weights proportional to each learner's accuracy.

B1.59

XGBoost

XGBoost (eXtreme Gradient Boosting) is a highly optimized gradient boosting implementation. Key innovations include:

  • Second‑order approximation: uses both gradient and Hessian for more accurate tree fitting.
  • Regularization: penalizes tree complexity (number of leaves, leaf weights) directly in the objective.
  • System optimizations: column block structure, cache‑aware access, out‑of‑core computation, parallelized tree construction.

XGBoost has dominated structured/tabular data competitions for years.

B1.60

LightGBM

LightGBM is a gradient boosting framework emphasizing speed and memory efficiency. Its key innovations:

  • Gradient‑based One‑Side Sampling (GOSS): keeps instances with large gradients (hard to fit) and samples those with small gradients.
  • Exclusive Feature Bundling (EFB): combines mutually exclusive sparse features to reduce dimensionality.
  • Leaf‑wise tree growth: grows the leaf with the highest loss reduction, producing asymmetric, deeper trees that can be more accurate than level‑wise growth.
B1.61

CatBoost

CatBoost is a gradient boosting library designed to handle categorical features natively — without manual one‑hot encoding. It uses ordered target encoding (with a permutation‑based scheme to avoid data leakage) and symmetric (oblivious) trees. It is known for strong out‑of‑the‑box performance, especially on datasets with many categorical columns.

B1.62

Stacking (Stacked Generalization)

Stacking combines multiple base models by training a meta‑learner on their outputs. Base models (level‑0) are trained on the original data; their predictions become features for the meta‑model (level‑1), which learns how to best combine them. To avoid overfitting, base model predictions should be generated via cross‑validation (out‑of‑fold predictions).

B1.63

Voting Classifiers

A voting classifier aggregates predictions from multiple heterogeneous models. Hard voting: each model casts a vote for a class; the majority wins. Soft voting: predicted class probabilities are averaged, and the class with highest average probability is selected. Soft voting often outperforms hard voting because it accounts for each model's confidence.

B1.64

Support Vector Machines (SVM)

A Support Vector Machine finds the hyperplane that maximizes the margin between classes. The decision boundary is determined entirely by the support vectors — the training examples closest to the hyperplane. SVMs are inherently regularized (maximizing margin controls complexity) and can learn non‑linear boundaries via the kernel trick.

B1.65

Maximum Margin Classifier

For linearly separable data, the maximum margin classifier solves:

minw,b ½∥w∥2 subject to yi(wTxi + b) ≥ 1, ∀i.

The margin is 2/∥w∥; minimizing ∥w∥ maximizes the margin. This is a convex quadratic program with a unique solution.

B1.66

Hard Margin SVM

The hard margin formulation requires perfect separation of the classes — no training point may violate the margin. This works only when the data is linearly separable. In practice, data is rarely separable; the soft‑margin formulation (B1.67) is preferred.

B1.67

Soft Margin SVM

The soft margin SVM allows some violations of the margin, controlled by slack variables ξi ≥ 0:

minw,b,ξ ½∥w∥2 + C ∑i ξi subject to yi(wTxi + b) ≥ 1 − ξi, ξi ≥ 0.

The hyperparameter C > 0 trades off margin maximization vs. training error tolerance. Large C: fewer violations, narrower margin (risk of overfitting). Small C: wider margin, more tolerance for misclassification (smoother boundary).

B1.68

Hinge Loss

The soft‑margin SVM objective can be written as an unconstrained empirical risk minimization with hinge loss:

ℓhinge(y, f̂(x)) = max(0, 1 − y·f̂(x)),

where y ∈ {−1,+1} and f̂(x) = wTx + b. The hinge loss is zero for correct predictions with margin ≥ 1, and grows linearly for margin violations. It is a convex surrogate for the 0‑1 loss.

B1.69

Kernel Trick

The kernel trick allows linear algorithms (SVMs, ridge regression, PCA) to learn non‑linear decision boundaries by implicitly mapping inputs into a high‑dimensional (possibly infinite‑dimensional) feature space, without ever computing the coordinates in that space. All computations are done via the kernel function K(x, x′) = ⟨φ(x), φ(x′)⟩, which computes the inner product in the feature space directly from the original inputs.

B1.70

Common Kernel Functions

  • Linear kernel: K(x, x′) = xTx′. No mapping; equivalent to standard linear SVM.
  • Polynomial kernel: K(x, x′) = (γ·xTx′ + r)d. Degree‑d polynomial decision boundaries.
  • RBF (Gaussian) kernel: K(x, x′) = exp(−γ∥x − x′∥2). Infinite‑dimensional feature space; the most popular non‑linear kernel.
  • Sigmoid kernel: K(x, x′) = tanh(γ·xTx′ + r). Inspired by neural networks.
B1.71

RBF (Gaussian) Kernel

The RBF kernel K(x, x′) = exp(−γ∥x − x′∥2) measures similarity as a Gaussian function of Euclidean distance. The parameter γ > 0 controls the "reach" of each training example: large γ → each point influences only its immediate neighborhood (complex, wiggly boundary, risk of overfitting); small γ → broad influence (smoother boundary). The RBF kernel is universal: its feature space is infinite‑dimensional and can approximate any continuous decision boundary.

B1.72

Polynomial Kernel

The polynomial kernel K(x, x′) = (γ·xTx′ + r)d corresponds to a feature space of all monomials up to degree d. The parameter r controls the relative weight of lower‑ vs. higher‑degree terms. Polynomial kernels are a natural choice when interactions between features are expected to be important, but the exponential growth of the feature space with d makes them impractical for very high degrees without the kernel trick.

B1.73

Mercer's Theorem

Mercer's Theorem characterizes valid kernel functions: a symmetric function K(x, x′) is a valid (Mercer) kernel — i.e., there exists a Hilbert space ℋ and a feature map φ such that K(x, x′) = ⟨φ(x), φ(x′)⟩ℋ — if and only if the Gram matrix [K(xi, xj)] is positive semidefinite for any finite set of points. This provides the mathematical foundation for the kernel trick.

B1.74

Support Vector Regression (SVR)

SVR extends SVM to regression. Instead of maximizing the margin between classes, SVR fits a tube of width ε around the regression line: errors smaller than ε are considered acceptable (zero loss). The objective minimizes ∥w∥2 (model flatness) while penalizing points outside the ε‑tube via the ε‑insensitive loss: ℓε(y, ŷ) = max(0, |y − ŷ| − ε). SVR produces sparse solutions where only points outside the tube are support vectors.

B1.75

One‑Class SVM

A One‑Class SVM is trained on only one class (the "normal" class) and learns a decision boundary that encloses the training data in feature space. It is used for anomaly detection: new points falling outside the boundary are flagged as anomalous. The hyperparameter ν controls the fraction of training points allowed to fall outside the boundary (an upper bound on the false positive rate).

B1.76

Dual Formulation of SVM

The SVM optimization is typically solved in its dual form, which expresses the problem in terms of Lagrange multipliers αi (one per training example). The dual has several advantages: (i) it depends on the data only through inner products xiTxj, enabling the kernel trick; (ii) it is a concave maximization problem (easier to solve); (iii) the solution is sparse — only support vectors have αi > 0. The dual is solved via Sequential Minimal Optimization (SMO) or quadratic programming.

B1.77

k‑Nearest Neighbors (k‑NN)

k‑NN is a non‑parametric, instance‑based algorithm. To predict for a new point x, it finds the k closest training examples (by some distance metric) and aggregates their labels: majority vote for classification, average for regression. k‑NN makes no explicit assumptions about the functional form; its inductive bias is local smoothness — nearby points are assumed to have similar outputs. The parameter k controls the bias–variance tradeoff: small k → low bias, high variance; large k → high bias, low variance.

B1.78

Distance Metrics

The choice of distance metric is crucial for k‑NN and other distance‑based methods. Common metrics:

  • Euclidean (L2): d(x, x′) = √∑(xj − x′j)2. Sensitive to scale differences.
  • Manhattan (L1): d(x, x′) = ∑|xj − x′j|. More robust to outliers in high dimensions.
  • Minkowski (Lp): generalizes L1 and L2.
  • Cosine distance: 1 − cos(θ). Ignores magnitude, captures directional similarity — popular for text data (TF‑IDF vectors).
  • Mahalanobis distance: accounts for feature correlations: d(x, x′) = √((x−x′)TΣ−1(x−x′)).
B1.79

Curse of Dimensionality

The curse of dimensionality refers to phenomena that arise in high‑dimensional spaces that make distance‑based methods (like k‑NN) break down: as dimensionality increases, (i) the volume of the space grows exponentially, making data increasingly sparse; (ii) pairwise distances tend to become uniform — the contrast between nearest and farthest neighbors diminishes. In practice, k‑NN becomes unreliable above ~10–20 dimensions unless the data lies on a lower‑dimensional manifold.

B1.80

Locally Weighted Regression (LOESS / LOWESS)

Locally Weighted Regression fits a (typically linear or quadratic) model using only the k nearest neighbors of the query point, with weights that decay with distance (e.g., a Gaussian kernel). This produces a smooth, non‑linear fit that adapts to local structure. It is a non‑parametric method; the entire training set must be stored, and predictions are computed on‑demand (lazy learning).

B1.81

Radial Basis Function (RBF) Networks

An RBF network is a feedforward neural network with a single hidden layer where each neuron uses an RBF activation (typically Gaussian). The output is a linear combination of these radial basis functions:

f̂(x) = ∑j=1m wj exp(−βj∥x − cj∥2).

Training typically involves two stages: (i) unsupervised selection of centers cj (e.g., via k‑means clustering), (ii) supervised learning of the output weights wj via linear regression.

B1.82

Naive Bayes Classifier

The Naive Bayes classifier applies Bayes' theorem with the "naive" assumption of conditional independence of features given the class:

P(y=k ∣ x) ∝ P(y=k) ∏j=1d P(x(j) ∣ y=k).

Despite the strong independence assumption (rarely true in practice), Naive Bayes often performs surprisingly well, especially for text classification. It is extremely fast, requires little data to estimate parameters, and provides well‑calibrated probability estimates.

B1.83

Bayes Theorem (Recap)

Bayes' theorem relates conditional and marginal probabilities:

P(y ∣ x) = P(x ∣ y) · P(y) / P(x).

In the classification context: P(y ∣ x) is the posterior (what we want), P(x ∣ y) is the likelihood (how features are distributed given the class), P(y) is the prior (class frequency), and P(x) is the evidence (normalizing constant).

B1.84

Gaussian Naive Bayes

When features are continuous, a common choice is to model P(x(j) ∣ y=k) as a Gaussian distribution: 𝒩(μkj, σkj2), where μkj and σkj2 are the mean and variance of feature j for class k, estimated from the training data. The decision boundary is quadratic (since the Gaussian likelihood yields a quadratic discriminant).

B1.85

Multinomial Naive Bayes

For discrete count data (especially text), Multinomial Naive Bayes models features as word counts drawn from a multinomial distribution per class. With Laplace smoothing (adding α to all counts to avoid zero probabilities), it is the classic baseline for document classification tasks like spam detection and topic categorization.

B1.86

Bayesian Linear Regression

In Bayesian linear regression, we place a prior distribution over the weights w (e.g., w ∼ 𝒩(0, τ2I)) and compute the posterior distribution P(w ∣ X, y) using Bayes' theorem. Prediction becomes: P(y* ∣ x*, X, y) = ∫ P(y* ∣ x*, w) P(w ∣ X, y) dw. This yields not just a point estimate but a full predictive distribution, capturing uncertainty. Under a Gaussian prior and Gaussian likelihood, the posterior is also Gaussian and has a closed form.

B1.87

Gaussian Processes (GP) for Regression

A Gaussian Process defines a distribution over functions. It is fully specified by a mean function m(x) and a covariance (kernel) function K(x, x′). Given training data, the posterior predictive distribution at a new point x* is Gaussian with closed‑form mean and variance. GPs provide principled uncertainty estimates and are highly flexible (via kernel choice), but scale as O(n3) with dataset size, limiting their application to moderate n (though sparse approximations exist).

B1.88

Confusion Matrix

A confusion matrix tabulates the counts of correct and incorrect predictions for each class. For binary classification:

TP (True Positive): predicted positive, actually positive
TN (True Negative): predicted negative, actually negative
FP (False Positive): predicted positive, actually negative (Type I error)
FN (False Negative): predicted negative, actually positive (Type II error)

The confusion matrix is the foundation for computing all classification metrics.

B1.89

Accuracy, Precision, and Recall

  • Accuracy: (TP + TN) / (TP + TN + FP + FN). Fraction of correct predictions overall. Misleading for imbalanced datasets.
  • Precision: TP / (TP + FP). Of all positive predictions, how many are correct? Measures exactness.
  • Recall (Sensitivity): TP / (TP + FN). Of all actual positives, how many are captured? Measures completeness.

There is an inherent tradeoff between precision and recall — improving one typically degrades the other.

B1.90

F1 Score

The F1 score is the harmonic mean of precision and recall:

F1 = 2 · (Precision · Recall) / (Precision + Recall).

The harmonic mean penalizes extreme imbalances (a model with perfect recall but near‑zero precision gets a low F1). The Fβ score generalizes this, weighting recall β times as important as precision. F1 is the standard single‑number metric when both false positives and false negatives matter.

B1.91

ROC Curve

The Receiver Operating Characteristic (ROC) curve plots the True Positive Rate (TPR = Recall) vs. the False Positive Rate (FPR = FP/(FP+TN)) as the classification threshold is varied. Each point on the curve corresponds to a different threshold. The ROC curve illustrates the diagnostic ability of a binary classifier across all thresholds.

B1.92

AUC (Area Under the ROC Curve)

AUC (or AUC‑ROC) is the area under the ROC curve, ranging from 0 to 1. It has a probabilistic interpretation: AUC is the probability that a randomly chosen positive example is ranked higher than a randomly chosen negative example. AUC = 0.5 corresponds to random guessing; AUC = 1.0 is perfect separation. AUC is threshold‑invariant and robust to class imbalance, making it a popular metric for comparing classifiers.

B1.93

Cross‑Validation

Cross‑validation (CV) is a resampling technique for estimating model performance and selecting hyperparameters. It partitions the data into k folds, trains on k−1 folds, tests on the held‑out fold, and repeats k times. The average test performance across folds provides a more reliable estimate than a single train/test split, at the cost of k-fold training time. CV is essential when data is limited.

B1.94

k‑Fold Cross‑Validation

In k‑fold CV, the data is randomly partitioned into k equal‑sized folds. Common choices are k = 5 or k = 10. Each fold serves as the validation set exactly once. The CV estimate is the mean performance across folds; its standard error provides a measure of uncertainty. Large k gives lower bias but higher variance and computational cost; k = n is Leave‑One‑Out CV (B1.95).

B1.95

Leave‑One‑Out Cross‑Validation (LOOCV)

LOOCV is the extreme case of k‑fold CV with k = n: each training set contains n−1 examples, and the held‑out set is a single example. LOOCV is nearly unbiased but can have high variance and is computationally expensive (requires n model fits). For linear models, LOOCV can be computed efficiently via closed‑form shortcuts (e.g., the PRESS statistic).

B1.96

Stratified k‑Fold Cross‑Validation

Stratified k‑fold CV preserves the class distribution in each fold — each fold has approximately the same proportion of each class as the full dataset. This is crucial for imbalanced datasets, where random splitting could produce folds with zero instances of a minority class, making evaluation meaningless. Stratification should be the default for classification tasks.

B1.97

Hyperparameter Tuning

Hyperparameters are parameters that control the learning process itself (e.g., λ in ridge regression, C and γ in SVM, learning rate, tree depth). Unlike model parameters, they are not learned from the training data but must be set before training. Hyperparameter tuning searches for the combination that minimizes validation error, typically using grid search, random search, or Bayesian optimization within a nested cross‑validation loop.

B1.98

Grid Search

Grid search exhaustively evaluates all combinations of hyperparameters from a pre‑specified grid. For m hyperparameters with ki values each, the total number of combinations is ∏ki, which grows exponentially. Grid search is simple, parallelizable, and guaranteed to find the best combination within the grid — but it is inefficient: many combinations may be uniformly bad, yet all are evaluated.

B1.99

Random Search

Random search samples hyperparameter combinations from specified distributions (uniform, log‑uniform, etc.) for a fixed budget of N trials. Empirically, random search often outperforms grid search for the same computational budget because (i) not all hyperparameters are equally important, and random search explores more distinct values of the important ones; (ii) it can discover good regions that a coarse grid might miss.

B1.100

Bayesian Optimization for Hyperparameter Tuning

Bayesian optimization builds a probabilistic surrogate model (typically a Gaussian Process) of the validation performance as a function of hyperparameters. It uses an acquisition function (e.g., Expected Improvement) to decide which hyperparameters to evaluate next, balancing exploration (trying uncertain regions) and exploitation (refining near the current best). Bayesian optimization is far more sample‑efficient than grid or random search, making it ideal when each evaluation is expensive.

B1.101

Learning Curves

A learning curve plots training and validation error as a function of training set size. It is a diagnostic tool:

  • High bias (underfitting): training and validation errors converge to a high value; more data won't help.
  • High variance (overfitting): large gap between training and validation error; more data is likely to help.
  • Well‑fit model: both errors are low and converge.

Learning curves guide decisions about collecting more data vs. changing model complexity.

B1.102

Early Stopping

Early stopping halts training when validation error stops improving (with a patience parameter to allow for temporary fluctuations). It is a simple, effective form of regularization: training a model beyond the point of minimum validation error leads to overfitting. Early stopping is standard practice when training neural networks and gradient‑boosted trees.

B1.103

Data Augmentation

Data augmentation artificially expands the training set by applying label‑preserving transformations to existing examples. For images: random crops, flips, rotations, color jittering. For text: synonym replacement, back‑translation. For tabular data: SMOTE (synthetic minority oversampling). Augmentation acts as a regularizer by teaching the model about valid invariances and increasing effective dataset size.

B1.104

Class Imbalance

Class imbalance occurs when one class (the majority) vastly outnumbers another (the minority). Standard algorithms trained on imbalanced data tend to ignore the minority class because doing so minimizes overall error. Strategies to address imbalance include:

  • Resampling: oversample the minority class, undersample the majority, or both.
  • Cost‑sensitive learning: assign higher misclassification cost to the minority class.
  • Synthetic data generation: SMOTE (B1.105).
  • Threshold adjustment: shift the decision threshold based on class priors.
  • Use metrics robust to imbalance: AUC, F1, precision‑recall AUC.
B1.105

SMOTE (Synthetic Minority Over‑sampling Technique)

SMOTE creates synthetic minority‑class examples by interpolating between existing minority examples. For each minority example, it selects k nearest minority neighbors and creates new points along the line segments connecting them. Unlike simple duplication, SMOTE produces genuinely new feature combinations, helping classifiers learn broader, less overfitted decision regions for the minority class.

B1.106

Feature Scaling

Feature scaling transforms features to a comparable range. It is essential for:

  • Distance‑based methods (k‑NN, SVM with RBF kernel, k‑means) — features with larger scales dominate distance calculations.
  • Gradient‑based optimization — features on different scales cause elongated loss contours, slowing convergence.
  • Regularized linear models — L1/L2 penalties are not scale‑invariant; coefficients of unscaled features are penalized unevenly.

Two common methods: standardization (B1.108) and normalization (B1.107).

B1.107

Min–Max Normalization

Min–max normalization rescales features to a fixed range, typically [0, 1]:

x′(j) = (x(j) − minj) / (maxj − minj).

It preserves zero values and the shape of the original distribution but is sensitive to outliers (which compress the majority of values into a narrow subrange). Best used when the data has known, bounded ranges (e.g., pixel intensities).

B1.108

Standardization (Z‑score Normalization)

Standardization centers each feature to zero mean and scales to unit variance:

x′(j) = (x(j) − μj) / σj.

It does not bound values to a fixed range, but is more robust to outliers than min–max normalization. Standardization is the default preprocessing step for most linear models, SVMs, and neural networks.

B1.109

One‑Hot Encoding

One‑hot encoding converts a categorical feature with c categories into c binary indicator columns. Exactly one column is "hot" (1) for each example; all others are 0. This avoids imposing an artificial ordinal relationship that would exist if categories were encoded as integers (e.g., "red"=1, "blue"=2, "green"=3). For high‑cardinality categorical features, alternative encodings (target encoding, embedding layers) are preferred.

B1.110

Feature Selection

Feature selection identifies and retains the most informative features while discarding irrelevant or redundant ones. Benefits include: simpler, more interpretable models; reduced overfitting; faster training and inference. Three main approaches:

  • Filter methods: rank features by a statistical score (correlation, mutual information, chi‑squared) independent of the model.
  • Wrapper methods: evaluate subsets of features by training a model (e.g., recursive feature elimination).
  • Embedded methods: feature selection is built into the training algorithm (e.g., Lasso's L1 penalty, tree feature importance).
B1.111

Dimensionality Reduction

Dimensionality reduction transforms the original high‑dimensional feature space into a lower‑dimensional one while preserving as much relevant information as possible. Unlike feature selection (which keeps a subset of original features), dimensionality reduction creates new features that are combinations of the originals. Principal techniques include PCA, t‑SNE, UMAP, and autoencoders.

B1.112

PCA as Preprocessing for Supervised Learning

Principal Component Analysis (PCA) projects data onto the directions of maximum variance. As preprocessing for supervised learning, PCA can reduce noise and accelerate training by eliminating near‑constant or highly correlated dimensions. However, PCA is unsupervised — the directions of maximum variance are not necessarily the directions most predictive of the target. Partial Least Squares (PLS) addresses this by finding components that maximize covariance with the target.

B1.113

Machine Learning Pipeline

A pipeline chains multiple processing steps (scaling, encoding, imputation, feature selection, model training) into a single object. Key benefits: (i) prevents data leakage — all transformations are fit only on training data and applied consistently to validation/test; (ii) simplifies cross‑validation and hyperparameter tuning; (iii) ensures reproducibility. Pipelines are a foundational engineering pattern in production ML.

B1.114

Probability Calibration

A classifier is calibrated if its predicted probability ŷ matches the empirical frequency: among examples where the model predicts probability p, a fraction p actually belong to the positive class. Many models (SVMs, boosted trees, Naive Bayes) produce poorly calibrated probabilities. Calibration methods include Platt scaling (fitting a logistic regression on raw scores) and isotonic regression (fitting a non‑decreasing function).

B1.115

Reliability Diagrams (Calibration Plots)

A reliability diagram bins predicted probabilities and plots the mean predicted probability vs. the observed fraction of positives in each bin. A perfectly calibrated model follows the diagonal. Deviations above the diagonal indicate under‑confidence; below the diagonal indicates over‑confidence. The Expected Calibration Error (ECE) summarizes the diagram as a weighted average of the bin gaps.

B1.116

Multi‑label Classification

In multi‑label classification, each example can belong to multiple classes simultaneously (unlike multi‑class, where classes are mutually exclusive). Example: tagging an image with multiple objects. Approaches include:

  • Binary relevance: train one binary classifier per label.
  • Classifier chains: train binary classifiers sequentially, using previous predictions as additional features.
  • Label powerset: treat each unique label combination as a separate class (exponential in number of labels).
  • Adapted algorithms: modifications of k‑NN, decision trees, and neural networks (with sigmoid outputs and binary cross‑entropy per label).
B1.117

Multi‑task Learning

Multi‑task learning trains a single model to perform multiple related tasks simultaneously, sharing representations across tasks. The shared layers learn features that are useful for all tasks, which acts as a powerful regularizer — it biases the model toward representations that capture underlying structure common to the tasks, improving generalization on each individual task (especially when data for some tasks is scarce).

B1.118

Transfer Learning

Transfer learning reuses a model trained on a large source task (e.g., ImageNet classification) as the starting point for a target task with limited data. Common approach: take a pre‑trained network, replace the final layer(s) with task‑specific ones, and fine‑tune — train with a small learning rate, possibly freezing early layers. Transfer learning is the dominant paradigm in computer vision and NLP, enabling state‑of‑the‑art performance with modest target‑domain data.

B1.119

Semi‑supervised Learning

Semi‑supervised learning leverages a small amount of labeled data together with a large amount of unlabeled data. Key assumptions that make this possible: (i) cluster assumption — points in the same cluster likely share a label; (ii) low‑density separation — the decision boundary should lie in low‑density regions; (iii) manifold assumption — the data lies on a low‑dimensional manifold. Techniques include self‑training, co‑training, and consistency regularization.

B1.120

Active Learning

Active learning is a paradigm where the learning algorithm can query an oracle (e.g., a human annotator) to label selected examples. The goal is to achieve high accuracy with minimal labeling cost by choosing the most informative examples. Query strategies include:

  • Uncertainty sampling: query the example the model is most uncertain about.
  • Query‑by‑committee: query where an ensemble of models disagrees most.
  • Expected model change: query the example that would most change the model if its label were known.
B1.121

Online Learning

In online learning, data arrives sequentially and the model is updated after each example (or small batch). There is no fixed training set; the model must continuously adapt. Key considerations: (i) the data distribution may shift over time (concept drift); (ii) computational efficiency is paramount. Algorithms like the perceptron, stochastic gradient descent, and online gradient boosting are natural fits. Online learning is essential for recommender systems, fraud detection, and high‑frequency trading.

B1.122

Curriculum Learning

Curriculum learning trains a model by presenting examples in a meaningful order — typically from "easy" to "hard" — rather than randomly. Inspired by human education, it can accelerate convergence and guide the model toward better local minima. The definition of "easy" is task‑dependent: shorter sentences in NLP, cleaner images in vision, lower‑noise examples. The approach can be seen as a form of continuation method in optimization.

B1.123

Imbalanced Learning (Extended)

Building on B1.104, imbalanced learning encompasses a broader set of techniques: ensemble methods (Balanced Random Forest, EasyEnsemble), anomaly detection approaches (treating the minority class as anomalies), threshold moving (adjusting the decision threshold post‑training based on class priors or cost ratios), and specialized loss functions (focal loss, which down‑weights easy examples and focuses on hard ones).

B1.124

Cost‑sensitive Learning

Cost‑sensitive learning incorporates different costs for different types of errors. In medical diagnosis, a false negative (missing a disease) may be far more costly than a false positive (unnecessary follow‑up). Costs can be integrated into the loss function (weighted loss), into resampling strategies, or via post‑hoc threshold adjustment. The goal shifts from minimizing error to minimizing expected cost.

B1.125

Explainability in Supervised Models

Explainability (or interpretability) concerns understanding why a model made a particular prediction. Methods are categorized as:

  • Intrinsic: the model is inherently interpretable (linear regression coefficients, decision tree rules).
  • Post‑hoc: explanations are generated after training, for any model type. Includes feature importance, SHAP (B1.126), LIME (B1.127), partial dependence plots (B1.128), and saliency maps.

Explainability is critical for trust, debugging, regulatory compliance, and fairness auditing.

B1.126

SHAP (SHapley Additive exPlanations)

SHAP values are a unified framework for interpreting model predictions based on Shapley values from cooperative game theory. The SHAP value of a feature is the average contribution of that feature to the prediction, averaged over all possible feature orderings. SHAP satisfies desirable axioms: local accuracy, missingness, and consistency. It provides both local explanations (per‑prediction) and global insights (via aggregation).

B1.127

LIME (Local Interpretable Model‑agnostic Explanations)

LIME explains individual predictions by locally approximating the complex model with a simple, interpretable model (e.g., linear regression) around the neighborhood of the instance being explained. It generates perturbed samples around the instance, gets predictions from the black‑box model, and fits an interpretable model weighted by proximity. LIME is model‑agnostic — it works with any classifier.

B1.128

Partial Dependence Plots (PDP)

A Partial Dependence Plot shows the marginal effect of a feature (or two features) on the predicted outcome, averaged over the distribution of all other features. For feature j:

PDPj(x) = (1/n) ∑i=1n f̂(xi(1), …, x, …, xi(d)).

PDPs reveal whether the relationship is linear, monotonic, or more complex. Their limitation is that they assume feature independence, which can produce misleading results for correlated features.

B1.129

Ethical Considerations in Supervised Learning

Supervised learning models can perpetuate and amplify biases present in training data. Key concerns:

  • Fairness: models should not discriminate based on protected attributes (race, gender, age). Metrics include demographic parity, equalized odds, and individual fairness.
  • Accountability: who is responsible when a model makes a harmful decision?
  • Transparency: stakeholders should understand how decisions are made.
  • Privacy: models may memorize training data; differential privacy provides formal guarantees.

Ethical ML is not merely a technical challenge — it requires interdisciplinary collaboration and ongoing societal dialogue.

B1.130

Summary of Supervised Learning

Supervised learning is the cornerstone of applied machine learning. The key themes of this reference:

  • Fundamentals: empirical risk minimization, bias–variance tradeoff, generalization.
  • Linear models: OLS, ridge, lasso, logistic regression — simple, interpretable, and often surprisingly effective.
  • Tree ensembles: random forests and gradient boosting — the dominant methods for tabular data.
  • Kernel methods: SVMs — elegant theory, strong performance on medium‑sized datasets.
  • Neural networks: MLPs — universal approximators that scale to massive data.
  • Probabilistic methods: Naive Bayes, Gaussian Processes — principled uncertainty quantification.
  • Evaluation: a rich suite of metrics, cross‑validation, and diagnostic tools (learning curves, calibration plots).
  • Advanced topics: transfer learning, multi‑task learning, active learning, explainability, and ethics.

Mastery of supervised learning requires both theoretical understanding and practical experience — knowing which algorithm to apply, how to preprocess data, how to tune hyperparameters, and how to rigorously evaluate results.

The field continues to evolve rapidly, but the foundational principles laid out in this reference remain constant — they provide the conceptual toolkit for understanding and applying any supervised learning method.
