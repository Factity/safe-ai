Machine learning is the study of algorithms that improve automatically through experience. This reference presents ten cornerstone concepts, each defined mathematically and accompanied by the relevant formulas and a brief discussion. All equations are rendered with MathJax.
A4.1

Linear Regression ML

Given a dataset $\{(\mathbf{x}_i, y_i)\}_{i=1}^n$ with $\mathbf{x}_i \in \mathbb{R}^d$, linear regression assumes a linear relationship $y_i \approx \mathbf{x}_i^\top \boldsymbol{\beta} + \beta_0$. The ordinary least squares (OLS) estimator minimizes the residual sum of squares:

\[ \hat{\boldsymbol{\beta}} = \arg\min_{\boldsymbol{\beta}, \beta_0} \sum_{i=1}^n \bigl(y_i - \mathbf{x}_i^\top \boldsymbol{\beta} - \beta_0\bigr)^2. \]

In matrix form with $\mathbf{X} \in \mathbb{R}^{n \times d}$ (including a column of ones for the intercept) and $\mathbf{y} \in \mathbb{R}^n$, the closed‑form solution is

\[ \hat{\boldsymbol{\beta}} = (\mathbf{X}^\top \mathbf{X})^{-1} \mathbf{X}^\top \mathbf{y}, \]

provided $\mathbf{X}^\top \mathbf{X}$ is invertible. The prediction for a new point $\mathbf{x}$ is $\hat{y} = \mathbf{x}^\top \hat{\boldsymbol{\beta}}$.

  • $\mathbf{X}$design matrix (each row is a sample, each column a feature).
  • $(\mathbf{X}^\top\mathbf{X})^{-1}\mathbf{X}^\top$Moore–Penrose pseudoinverse of $\mathbf{X}$.
A4.2

Logistic Regression ML

For binary classification ($y_i \in \{0,1\}$), logistic regression models the probability $P(Y=1 \mid \mathbf{x})$ using the sigmoid function:

\[ \sigma(z) = \frac{1}{1+e^{-z}}, \qquad z = \mathbf{x}^\top \boldsymbol{\beta} + \beta_0. \]

The parameters are estimated by maximizing the likelihood (or minimizing the negative log‑likelihood), which leads to the cross‑entropy loss:

\[ \mathcal{L}(\boldsymbol{\beta}) = -\sum_{i=1}^n \Bigl[ y_i \log \sigma(\mathbf{x}_i^\top \boldsymbol{\beta}) + (1-y_i) \log\bigl(1-\sigma(\mathbf{x}_i^\top \boldsymbol{\beta})\bigr) \Bigr]. \]

Gradient descent or Newton’s method is used because no closed‑form solution exists.

  • $\sigma(z)$logistic sigmoid; maps $\mathbb{R} \to (0,1)$.
A4.3

Regularization: Ridge and Lasso ML

To prevent overfitting, a penalty is added to the loss. Ridge regression uses $\ell_2$ regularization:

\[ \hat{\boldsymbol{\beta}}_{\text{ridge}} = \arg\min_{\boldsymbol{\beta}} \Bigl\{ \|\mathbf{y} - \mathbf{X}\boldsymbol{\beta}\|_2^2 + \lambda \|\boldsymbol{\beta}\|_2^2 \Bigr\}, \]

with closed‑form solution $(\mathbf{X}^\top \mathbf{X} + \lambda \mathbf{I})^{-1} \mathbf{X}^\top \mathbf{y}$.

Lasso uses $\ell_1$ regularization, which promotes sparsity:

\[ \hat{\boldsymbol{\beta}}_{\text{lasso}} = \arg\min_{\boldsymbol{\beta}} \Bigl\{ \|\mathbf{y} - \mathbf{X}\boldsymbol{\beta}\|_2^2 + \lambda \|\boldsymbol{\beta}\|_1 \Bigr\}. \]

Lasso does not have a closed form and is typically solved via coordinate descent or proximal gradient methods.

  • $\lambda > 0$regularization strength; larger $\lambda$ shrinks coefficients more.
A4.4

Support Vector Machines ML

For binary classification with labels $y_i \in \{-1, +1\}$, the linear SVM finds the maximum‑margin separating hyperplane by solving

\[ \min_{\mathbf{w}, b} \frac12 \|\mathbf{w}\|^2 \quad \text{s.t. } y_i(\mathbf{w}^\top \mathbf{x}_i + b) \ge 1,\; i=1,\dots,n. \]

For non‑separable data, slack variables $\xi_i$ are introduced (soft margin):

\[ \min_{\mathbf{w}, b, \boldsymbol{\xi}} \frac12 \|\mathbf{w}\|^2 + C \sum_{i=1}^n \xi_i \quad \text{s.t. } y_i(\mathbf{w}^\top \mathbf{x}_i + b) \ge 1 - \xi_i,\; \xi_i \ge 0. \]

The dual formulation allows the use of kernel functions $K(\mathbf{x}_i, \mathbf{x}_j)$ for non‑linear classification.

  • $C$regularization parameter trading off margin width and misclassification.
A4.5

Decision Trees ML

A decision tree recursively partitions the feature space into axis‑aligned rectangles. At each node, a split $(j, s)$ (feature $j$, threshold $s$) is chosen to maximize the reduction in impurity:

\[ \Delta \mathcal{I} = \mathcal{I}(\text{parent}) - \frac{N_{\text{left}}}{N} \mathcal{I}(\text{left}) - \frac{N_{\text{right}}}{N} \mathcal{I}(\text{right}). \]

Common impurity measures include Gini impurity $\mathcal{I}(t) = \sum_{k=1}^K p_k(1-p_k)$ and entropy $\mathcal{I}(t) = -\sum_{k=1}^K p_k \log p_k$.

Single trees tend to overfit; random forests and gradient boosting combine many trees to improve generalization.

  • $p_k$proportion of samples in node $t$ belonging to class $k$.
A4.6

Ensemble Methods ML

Ensembles combine multiple weak learners to produce a strong learner. Two major families:

  • Bagging (Bootstrap Aggregating): trains each model on a bootstrap sample of the data and averages predictions (e.g., Random Forest). Reduces variance.
  • Boosting: sequentially trains models, each focusing on the mistakes of the previous one. AdaBoost weights misclassified samples more heavily; Gradient Boosting fits each new model to the negative gradient of the loss.

The final prediction for a regression ensemble is $\hat{f}(\mathbf{x}) = \frac{1}{M}\sum_{m=1}^M f_m(\mathbf{x})$ (bagging) or a weighted sum $\sum_{m=1}^M \alpha_m f_m(\mathbf{x})$ (boosting).

A4.7

Neural Networks (Feedforward) ML

A feedforward neural network is a composition of affine transformations and non‑linear activation functions. A layer $\ell$ computes:

\[ \mathbf{z}^{(\ell)} = \mathbf{W}^{(\ell)} \mathbf{a}^{(\ell-1)} + \mathbf{b}^{(\ell)}, \qquad \mathbf{a}^{(\ell)} = \sigma(\mathbf{z}^{(\ell)}), \]

where $\mathbf{a}^{(0)} = \mathbf{x}$ is the input, and $\sigma$ is an activation such as ReLU $\sigma(z)=\max(0,z)$, sigmoid, or tanh. The final layer typically uses a linear output for regression or softmax for classification:

\[ \text{softmax}(\mathbf{z})_k = \frac{e^{z_k}}{\sum_{j} e^{z_j}}. \]

The network is trained by minimizing a loss function using gradient‑based optimization (e.g., SGD) with gradients computed via backpropagation.

  • $\mathbf{W}^{(\ell)}, \mathbf{b}^{(\ell)}$weight matrix and bias vector of layer $\ell$.
A4.8

Backpropagation ML

Backpropagation efficiently computes the gradient of the loss $\mathcal{L}$ with respect to all parameters using the chain rule. Define the error signal $\boldsymbol{\delta}^{(\ell)} = \frac{\partial \mathcal{L}}{\partial \mathbf{z}^{(\ell)}}$. For layer $\ell$ (from output backwards):

\[ \boldsymbol{\delta}^{(\ell)} = \bigl(\mathbf{W}^{(\ell+1)}\bigr)^\top \boldsymbol{\delta}^{(\ell+1)} \odot \sigma'(\mathbf{z}^{(\ell)}), \]

and the parameter gradients are

\[ \frac{\partial \mathcal{L}}{\partial \mathbf{W}^{(\ell)}} = \boldsymbol{\delta}^{(\ell)} (\mathbf{a}^{(\ell-1)})^\top, \qquad \frac{\partial \mathcal{L}}{\partial \mathbf{b}^{(\ell)}} = \boldsymbol{\delta}^{(\ell)}. \]

One forward pass stores activations; one backward pass yields all gradients — crucial for training deep networks.

  • $\odot$element‑wise (Hadamard) product.
A4.9

Principal Component Analysis (PCA) ML

PCA reduces dimensionality by projecting data onto the directions of maximum variance. Given centered data $\mathbf{X} \in \mathbb{R}^{n \times d}$, compute the eigendecomposition of the covariance matrix:

\[ \mathbf{C} = \frac{1}{n-1} \mathbf{X}^\top \mathbf{X} = \mathbf{V} \boldsymbol{\Lambda} \mathbf{V}^\top, \]

where $\boldsymbol{\Lambda} = \operatorname{diag}(\lambda_1,\dots,\lambda_d)$ with $\lambda_1 \ge \cdots \ge \lambda_d$. The first $k$ principal components are the columns of $\mathbf{V}_k$ (the top $k$ eigenvectors). The transformed data is $\mathbf{Z} = \mathbf{X} \mathbf{V}_k$.

PCA can also be performed via the SVD of $\mathbf{X}$.

  • $\lambda_i$eigenvalue; variance explained by the $i$‑th principal component.
A4.10

K‑Means Clustering ML

K‑means partitions $n$ observations into $k$ clusters by minimizing the within‑cluster sum of squares:

\[ \min_{\{C_j\}_{j=1}^k} \sum_{j=1}^k \sum_{\mathbf{x} \in C_j} \|\mathbf{x} - \boldsymbol{\mu}_j\|^2, \quad \boldsymbol{\mu}_j = \frac{1}{|C_j|} \sum_{\mathbf{x} \in C_j} \mathbf{x}. \]

The standard algorithm (Lloyd’s algorithm) alternates between:

  1. Assignment step: assign each point to the nearest centroid.
  2. Update step: recompute centroids as the means of the assigned points.

Convergence is guaranteed to a local minimum; multiple restarts help find a good solution.

  • $\boldsymbol{\mu}_j$centroid (prototype) of cluster $j$.
